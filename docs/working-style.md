# Working Style

- Owner-facing progress and final responses should be concise Chinese,
  conclusion-first.
- Keep commands, paths, schema keys, status enums, repo names, model names, and
  secret aliases in English.
- Report decisions, evidence, blockers, and next actions only.
- Make reasonable assumptions for low-risk work.
- Ask only for dangerous actions, Owner-gated actions, or true blockers.
- Keep working until PASS or a real Owner-gated BLOCKED state.
- Do not claim done without evidence.
